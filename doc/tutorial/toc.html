Overview

The CUDA HPC tutorial contains a collection of programs, each more or less built atop of previous ones, which demonstrate various aspects of using CUDA to speed up numerical calculations.

A major issue is to encapsulate the CUDA parts of a program such that the effort to replace it by another parallelization technique is minimized. As CUDA itself is still subject to drastic changes this should simplify adaption to new CUDA features as well. Besides learning how to do parallel computations on a GPU the projects also have a strong focus on software design. Nowadays simulations become more and more complex and a good object-oriented design can help in reducing development times by making more parts of the code reusable.

Good object-oriented designs do not come overnight. Therefore, these projects will change with time. Due to the large number of projects not all of them will always be in sync with the latest features of the libraries referenced by these projects.

For people working with the great finite element library deal.II the layout of this online documentation will be very familiar.

Each project has a structure as follows:

  1. Introduction:A brief description of what the program does and what is new compared to the previous programs. This also includes a discussion of mathematical or other special topics and their formulation.
  2. The commented program: A listing of the program, interspersed with many comments that explain what is going on, how things are done and why.
  3. Results:The output of the program, with some comments and interpretation. For the more complex examples we also give a short introduction how to use the program.
  4. The plain program: Since the program itself usually is a little bit unreadable with all the comments intertwingled, we present the program again, with all comments stripped off, to allow getting an overview of it.

The programs can be found at a path which is indicated at the beginning of the last section of each example.

As IDE we favor QTCreator. Mainly because of its ease of use, OS-independence and the tight integration of QT. This is especially helpful when dealing with such basic things like string processing. QT provides a lot of tools for that making string-processing as simple as with scripting languages like python.

To compile and run a program, load the *.pro-file of the project into QtCreator and press the run button (that's the one with the green triangle on it). No bothering about making make files is necessary. Under the hood qmake does it all for you. All you have to do is to adapt the paths in the project file to your local CUDA and deal.II installation.

Below we will list the currently available tutorial programs twice, once in numerical order explaining what is new in each program and once listed by topic to facilitate searching for programs for certain functionality.

Tutorial programs listed by number

Step-1
This introductory example is not so much about CUDA but discusses the general layout of the following programs.

Step-2
The first project actually dealing with CUDA shows various to perform a matrix-vector product, one of the most important operations in scientific computing. We compare the variant by Fujimoto with ATLAS and CUBLAS.
As a by-product we get know some of the caching mechanisms CUDA offers: shared memory and textures.

Step-3
The other basic operation is the matrix-matrix product. In this program we discuss the special case of multiplying a matrix with its own transpose which reveals certain subtleties in CUDA's memory management. Especially how to deal with what is called a shared memory bank conflict.

Step-4
QR-Factorization using our BLAS wrapper with CUBLAS and ATLAS under the hood. The main feature is the use of expression templates for lazy evaluation. The performance of CUBLAS is benchamrked against a well tuned CPU-based blas implementation. Due to the template approach algorithms are identical on both platforms.

Step-5
This project investigates principal component analysis based on the NIPALS algorithm. It serves as testsite for measuring the runtime overhead of expression templates.

Step-6
As precursor for step-7 this example program is dedicated to cluster analysis algorithms and their parallelizability.

Step-7
This example combines the previous steps 4, 5 and 6 into an analysis tool for experiments of in vivo neural networks. The central tool is an algorithm by Shy Shoham published in 2009 for non-Gaussian mixture modelling based on the t-distribution. The highlight of the results are a movie which shows the algorithm in action.

Step-8
Here, we discuss how to compute the Cholesky factorization on a GPU. The program runs the factorization for a range of matrix sizes which are set in a parameter file. As an additional gimick we move the execution of the benchmark test into a separate QThread.

Step-9
The generalization of the Cholesky factorization is LU decomposition which will be discussed in this project. This is still on our TODO-list.

Step-12
Finite difference and multigrid methods for the Poisson-Boltzmann equation for accelerating Monte-Carlo simulations of simple models for protein folding. The Poisson-Boltzmann equation models the energy contribution of screening effects due to exterior counter ions. The intramolecular dynamics is simulated using the HP lattice model which will be discussed in a forthcoming project.

Step-16
In this project the Gerchberg-Saxton algorithm is used to calculate the phase-only hologram of a given intensity pattern. By displaying the hologram on a spatial light modulator the intensity patern can be reconstructed and usd for spatially selective stimulation of neural networks based on light-sensitive cells.

Step-16 GUI
As a follow-up of step-16 this project aims at integrating CUDA with QT in order to display the phase-only hologram by OpenGL via texture mapping.

Step-23
Faber polynomials for quntaum time propagation in extended 2D systems. The goal is to simulate the transport through magnetic focussing devices in time.

Step-28
Multigrid-FEM-BEM solver for dielectric interface problems. We use CUDA to accelerate the assembly of the BEM part of the matrix. The assembly of the FEM part of the matrix is parallelized using QThreads using a domain-decomposition-like approach. This program continues the works started in step-12 on the electrostatics of a protein.
